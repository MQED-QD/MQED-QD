1. What Is an SGE Script?

Sun Grid Engine (SGE) is a workload manager used on many HPC clusters (including Notre Dame's CRC). An SGE script is simply a Bash shell script with special comment lines (#$ …) that tell the scheduler how many resources you need, where to send email notifications, which queue to use, etc. The scheduler reads these directives, allocates the resources, and then runs the rest of the script as a normal Bash program on the assigned compute node(s).

Key insight Everything after the #$ directive block is ordinary Bash. SGE only cares about the lines starting with #$. The rest is your program.

2. SGE Directive Grammar

SGE directives follow this pattern:

#$ -<flag> [value]
  • #$ — the directive prefix. SGE scans lines that start with #$ (after the shebang). To Bash, these are just comments and are ignored.
  • -<flag> — a single-letter or long-form option (same flags you can pass to qsub on the command line).
  • [value] — the argument for that flag (some flags are boolean and take no value).
Tip Anything you put after #$ can also be passed on the qsub command line, and command-line values override the script directives. So #$ -t 1-4 in the script can be overridden by qsub -t 1-10 script.sh.

3. Directive-by-Directive Reference

Here are the directives used in our scripts:

#!/bin/bash
#$ -N angle_disorder_array
#$ -t 1-4
#$ -M gliu8@nd.edu
#$ -m abe
#$ -pe smp 24
#$ -q long@@theta_lab
#$ -cwd
Directive Flag Meaning
#!/bin/bash — Shebang. Not an SGE directive. Tells the OS to execute this file with /bin/bash. Must be the very first line.
#$ -N angle_disorder_array -N Job name. Shows up in qstat output and log filenames. Choose something short but descriptive.
#$ -t 1-4 -t Array task range. Launches tasks with IDs 1, 2, 3, 4. Each task gets its own $SGE_TASK_ID. This is the heart of a job array.
#$ -M gliu8@nd.edu -M Mail address. Where SGE sends notification emails.
#$ -m abe -m Mail events. a = abort, b = begin, e = end. You get an email when the job starts, finishes, or crashes.
#$ -pe smp 24 -pe Parallel environment. Requests 24 slots in the smp (shared-memory parallel) environment. SGE sets $NSLOTS=24. We pass this to mpirun -np.
#$ -q long@@theta_lab -q Queue. Submits to the long queue, restricted to the @@theta_lab host group. The @@ syntax selects a host group within the queue.
#$ -cwd -cwd Current working directory. Runs the job in the directory from which qsub was called (instead of your home directory). SGE sets $SGE_O_WORKDIR to this path.
Other useful directives (not in our scripts)
  • #$ -l h_rt=48:00:00 — wall-clock time limit
  • #$ -l mem_free=8G — minimum free memory per slot
  • #$ -o /path/to/stdout.log — redirect stdout
  • #$ -e /path/to/stderr.log — redirect stderr
  • #$ -j y — merge stderr into stdout
  • #$ -V — export all current environment variables to the job

4. Full Annotated Walkthrough

Below is angle_disorder_jobarray.sh broken into logical blocks with explanations. The same structure applies to jobarray.sh (NN-chain variant) — only the parameter names and the final command differ.

Block 1 — Shebang & SGE Directives

#!/bin/bash
#$ -N angle_disorder_array
#$ -t 1-4
#$ -M gliu8@nd.edu
#$ -m abe
#$ -pe smp 24
#$ -q long@@theta_lab
#$ -cwd

Covered in Section 3. These lines are metadata for the scheduler. Bash ignores them.

Block 2 — Strict Bash Mode

set -euo pipefail

Activates three safety nets:

FlagMeaningWhy it matters
-e Exit immediately if any command returns non-zero Prevents the script from silently continuing after a failure
-u Treat unset variables as errors Catches typos like $TAKS_INDEX instead of $TASK_INDEX
-o pipefail A pipeline fails if any stage fails (not just the last) Without this, bad_cmd | good_cmd reports success
Always use this in HPC scripts. A silent failure wastes your allocation hours — the job keeps running (and billing) while doing nothing useful.

Block 3 — Conda Environment Activation

module load conda
source "$(conda info --base)/etc/profile.d/conda.sh"
conda activate mqed

HPC compute nodes start with a minimal environment. module load conda makes the conda command available. The source line initializes conda's shell integration (so that conda activate works instead of requiring source activate). Finally we activate the mqed environment that has all our Python dependencies.

Block 4 — Repository Root Detection

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
if [ -n "${MQED_REPO_ROOT:-}" ]; then
  REPO_ROOT="${MQED_REPO_ROOT}"
elif [ -n "${SGE_O_WORKDIR:-}" ] && [ -f "${SGE_O_WORKDIR}/mqed/disorder/angle_sweep_params.tsv" ]; then
  REPO_ROOT="${SGE_O_WORKDIR}"
else
  REPO_ROOT="$(cd "${SCRIPT_DIR}/../.." && pwd)"
fi
cd "${REPO_ROOT}"

We need to know where the repository lives so we can find the TSV parameter file and set the working directory. The script tries three strategies in priority order:

  1. MQED_REPO_ROOT — explicitly passed via qsub -v MQED_REPO_ROOT=/path/to/repo. Most reliable.
  2. SGE_O_WORKDIR — the directory from which qsub was called (set automatically by SGE when you use #$ -cwd). We sanity-check that the TSV file exists there.
  3. Relative to script location — fallback for local testing. BASH_SOURCE[0] is the path to the script itself; going up two directories reaches the repo root.
The ${VAR:-} syntax means "use $VAR if set, otherwise use empty string". This prevents set -u from aborting on an unset variable. It's a Bash parameter expansion with default.

Block 5 — Task Index & Core Variables

TASK_INDEX="${SGE_TASK_ID:-${1:-1}}"
MPI_NPROC="${NSLOTS:-24}"
CONFIG_NAME="${ANGLE_CONFIG_NAME:-my_angle_disorder}"

Three key variables with layered defaults:

VariablePrimary sourceFallbackLast resort
TASK_INDEX $SGE_TASK_ID
(set by SGE for each array task)		 $1
(first CLI argument for local testing)		 1
MPI_NPROC $NSLOTS
(set by SGE from -pe smp 24)		 24
CONFIG_NAME $ANGLE_CONFIG_NAME
(user-set env var)		 my_angle_disorder

The nested default pattern ${A:-${B:-C}} reads as: "use A if set; otherwise use B if set; otherwise use C."

Block 6 — Parameter File Resolution

DEFAULT_PARAM_FILE="mqed/disorder/angle_sweep_params.tsv"
PARAM_FILE_INPUT="${ANGLE_SWEEP_PARAM_FILE:-${MQED_SWEEP_PARAM_FILE:-${2:-${DEFAULT_PARAM_FILE}}}}"

if [[ "${PARAM_FILE_INPUT}" = /* ]]; then
  PARAM_FILE="${PARAM_FILE_INPUT}"
else
  PARAM_FILE="${REPO_ROOT}/${PARAM_FILE_INPUT}"
fi

if [ ! -f "${PARAM_FILE}" ]; then
  echo "Parameter file not found: ${PARAM_FILE}" >&2
  exit 1
fi

Resolves the TSV file path with a priority chain: ANGLE_SWEEP_PARAM_FILE → MQED_SWEEP_PARAM_FILE → second CLI argument → default path. Then handles both absolute (/home/...) and relative paths. Exits early with an error message if the file doesn't exist.

Tip: >&2 redirects output to stderr, keeping error messages separate from normal output. On SGE, stdout and stderr go to separate log files by default.

Block 7 — TSV Parsing with awk

PARAM_LINE="$(awk -v FS='[[:space:]]+' -v OFS='\t' -v idx="${TASK_INDEX}" '
  NR > 1 && NF {
    count++
    if (count == idx) {
      if (NF != 7) {
        print "__MALFORMED__", NF, $0
        exit
      }
      print $1, $2, $3, $4, $5, $6, $7
      exit
    }
  }
' "${PARAM_FILE}")"

This awk one-liner is the bridge between the SGE task ID and the parameter file. Here is how it works:

ElementMeaning
-v FS='[[:space:]]+' Field separator: one or more whitespace characters (tabs or spaces)
-v OFS='\t' Output field separator: tab (so output is clean TSV)
-v idx="${TASK_INDEX}" Pass the Bash variable into awk as idx
NR > 1 Skip the header row (line 1)
NF Skip blank lines (NF = number of fields; 0 is falsy)
count++ Count data rows; when count == idx, this is our row
NF != 7 Validation: each row must have exactly 7 fields

The output is a single tab-separated line like:
baseline<TAB>5.0<TAB>45.0<TAB>200<TAB>1.0<TAB>0.0<TAB>3.0

Block 8 — Validation & Destructuring

if [ -z "${PARAM_LINE}" ]; then
  echo "No parameter row found for task index ${TASK_INDEX}" >&2
  exit 1
fi

if [[ "${PARAM_LINE}" == __MALFORMED__* ]]; then
  BAD_FIELD_COUNT="$(printf '%s' "${PARAM_LINE}" | cut -f2)"
  echo "Malformed row: expected 7 fields, found ${BAD_FIELD_COUNT}" >&2
  exit 1
fi

IFS=$'\t' read -r LABEL SIGMA_PHI PHI NMOL D_NM KPARALLEL SIGMA_SITES <<< "${PARAM_LINE}"

Three checks happen here:

  1. Empty result — the task index exceeds the number of rows (e.g., -t 1-10 but only 4 data rows).
  2. Malformed sentinel — awk printed __MALFORMED__ because a row had the wrong number of columns.
  3. Destructuring — the read command splits the tab-separated line into named variables. IFS=$'\t' sets the delimiter to tab for this one command. -r prevents backslash interpretation.

Block 9 — Diagnostic Logging

echo "Running task ${TASK_INDEX} with label=${LABEL}"
echo "Parameter file: ${PARAM_FILE}"
echo "Config name: ${CONFIG_NAME}"
echo "sigma_phi_deg=${SIGMA_PHI}, phi_deg=${PHI}, Nmol=${NMOL}, d_nm=${D_NM}"
echo "k_parallel=${KPARALLEL}, sigma_sites=${SIGMA_SITES}"
echo "Using ${MPI_NPROC} MPI ranks"
echo "start time: $(date)"

Prints all resolved parameters and a timestamp to stdout. These go to the SGE log file (usually angle_disorder_array.o<JOB_ID>.<TASK_ID>). Invaluable for debugging failed jobs after the fact.

Block 10 — The Actual Computation

mpirun -np "${MPI_NPROC}" mqed_nhse_disorder \
  --config-name "${CONFIG_NAME}" \
  disorder.backend=mpi \
  disorder.mpi_auto_launch=false \
  disorder.mpi_nproc="${MPI_NPROC}" \
  simulation.disorder_sigma_phi_deg="${SIGMA_PHI}" \
  simulation.phi_deg="${PHI}" \
  simulation.Nmol="${NMOL}" \
  simulation.d_nm="${D_NM}" \
  initial_state.k_parallel="${KPARALLEL}" \
  initial_state.sigma_sites="${SIGMA_SITES}" \
  hydra.run.dir="outputs/angle_sweep/${LABEL}"

echo "end time: $(date)"

This is where the physics happens. Let's break it down:

PartMeaning
mpirun -np "${MPI_NPROC}" Launch the program with N MPI processes (one per SGE slot)
mqed_nhse_disorder The Python entry point (installed via pip install -e .)
--config-name "${CONFIG_NAME}" Hydra flag: which YAML config to load (e.g., my_angle_disorder)
disorder.backend=mpi Hydra override: use MPI backend instead of joblib
disorder.mpi_auto_launch=false We already launched with mpirun; don't let Python re-launch
simulation.disorder_sigma_phi_deg=... Hydra override: inject the sweep parameter from the TSV row
hydra.run.dir="outputs/angle_sweep/${LABEL}" Per-task output directory, named by the label column
Hydra CLI overrides Every key=value after the config name is a Hydra override. They override values from the YAML config file, giving you full control from the command line without editing YAML.
Why mpi_auto_launch=false? The Python code has its own mpirun auto-launch logic for convenience when running locally. On HPC, SGE already manages the MPI launch, so we disable the Python-level auto-launch to avoid double-spawning.

5. How Job Arrays Work

A job array is SGE's way of running the same script many times with different inputs, without submitting separate jobs.

#$ -t 1-4          ←  creates tasks with IDs: 1, 2, 3, 4
                       each task runs the same script independently

The lifecycle:

  1. You submit one script with qsub -t 1-4 script.sh.
  2. SGE creates 4 independent tasks, each with a unique $SGE_TASK_ID (1, 2, 3, or 4).
  3. Each task may run on a different node at a different time.
  4. Inside the script, you use $SGE_TASK_ID to select the row from the TSV file, so each task gets different parameters.

           qsub -t 1-4 angle_disorder_jobarray.sh
                        |
           SGE creates 4 independent tasks:
           ┌─────────────────────────────────────────────────┐
           │  Task 1 (SGE_TASK_ID=1)  ─→  TSV row 1         │
           │  Task 2 (SGE_TASK_ID=2)  ─→  TSV row 2         │
           │  Task 3 (SGE_TASK_ID=3)  ─→  TSV row 3         │
           │  Task 4 (SGE_TASK_ID=4)  ─→  TSV row 4         │
           └─────────────────────────────────────────────────┘
           Each task runs on its own node(s) with 24 MPI ranks
Step ranges You can also use -t 1-100:5 to get task IDs 1, 6, 11, 16, … (step of 5). Useful for coarser sweeps.

6. Important SGE Environment Variables

VariableSet byMeaning
SGE_TASK_ID SGE (array jobs) Current task index within the array (1, 2, 3, …)
NSLOTS SGE (-pe) Number of slots (CPU cores) allocated to this job
SGE_O_WORKDIR SGE (-cwd) Directory from which qsub was called
JOB_ID SGE Unique numeric job ID assigned by the scheduler
JOB_NAME SGE (-N) Job name from the -N directive
HOSTNAME SGE Compute node hostname where the task is running
TMPDIR SGE Per-job temporary directory (fast local storage on the node)

7. Submitting & Controlling Jobs

Submission

# Submit with defaults (uses #$ directives in the script)
qsub angle_disorder_jobarray.sh

# Override the array range and pass an env variable
qsub -t 1-10 -v ANGLE_CONFIG_NAME=quantum_dynamics_disorder angle_disorder_jobarray.sh

# Use the convenience wrapper (counts TSV rows automatically)
bash submit_angle_sweep.sh

Monitoring

# List your jobs
qstat

# List only array tasks for a specific job
qstat -j <JOB_ID>

# Watch jobs refresh every 5 seconds
watch -n 5 qstat

Cancellation

# Cancel an entire job array
qdel <JOB_ID>

# Cancel a specific task within an array
qdel <JOB_ID>.<TASK_ID>

# Cancel all your jobs
qdel -u $USER

Log files

SGE writes stdout and stderr to files in the submission directory (because of #$ -cwd): 

angle_disorder_array.o<JOB_ID>.<TASK_ID>    ← stdout
angle_disorder_array.e<JOB_ID>.<TASK_ID>    ← stderr

8. SGE vs. SLURM Cheat Sheet

Many clusters use SLURM instead of SGE. Here is a side-by-side comparison so you can translate between the two systems:

Concept SGE SLURM
Directive prefix #$ #SBATCH
Submit a job qsub script.sh sbatch script.sh
Job name #$ -N myname #SBATCH --job-name=myname
Array tasks #$ -t 1-100 #SBATCH --array=1-100
Task ID variable $SGE_TASK_ID $SLURM_ARRAY_TASK_ID
Number of cores #$ -pe smp 24 #SBATCH --ntasks=24
Cores variable $NSLOTS $SLURM_NTASKS
Queue / Partition #$ -q long #SBATCH --partition=long
Wall time #$ -l h_rt=48:00:00 #SBATCH --time=48:00:00
Email #$ -M user@nd.edu
#$ -m abe		 #SBATCH --mail-user=user@nd.edu
#SBATCH --mail-type=ALL
Working directory #$ -cwd #SBATCH --chdir=/path
(defaults to submission dir)
Check status qstat squeue -u $USER
Cancel job qdel <ID> scancel <ID>
Pass env vars qsub -v KEY=VAL sbatch --export=KEY=VAL
